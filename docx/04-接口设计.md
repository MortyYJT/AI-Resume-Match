# 接口设计

这份文档给前后端对接口用。第一期只实现两个接口。

## 通用响应格式

所有接口统一返回：

```json
{
  "code": 0,
  "message": "success",
  "data": {}
}
```

失败时：

```json
{
  "code": 400,
  "message": "错误原因",
  "data": null
}
```

## 1. 健康检查接口

用途：确认后端启动成功。

```text
GET /api/health
```

响应：

```json
{
  "code": 0,
  "message": "success",
  "data": {
    "status": "ok"
  }
}
```

## 2. Mock 匹配分析接口

用途：第一期给前端联调用，先不接真实 AI。

```text
POST /api/matches/mock
```

请求：

```json
{
  "resumeText": "这里是简历内容",
  "jdText": "这里是岗位 JD"
}
```

响应：

```json
{
  "code": 0,
  "message": "success",
  "data": {
    "score": 82,
    "level": "高",
    "advantages": [
      "Java 基础较完整",
      "项目经历和岗位方向接近"
    ],
    "gaps": [
      "缺少 MySQL 实战描述",
      "缺少部署经验"
    ],
    "resumeSuggestions": [
      "补充项目中的接口设计和数据库表设计",
      "强化项目难点和个人贡献"
    ],
    "interviewSuggestions": [
      "准备 Spring Boot、MySQL、接口设计相关问题"
    ],
    "deliverySuggestion": "可以投递，但建议先补强项目描述"
  }
}
```

## 前端第一期只需要接这个流程

```text
用户输入 resumeText 和 jdText
点击开始分析
请求 POST /api/matches/mock
拿到 data
展示 score、level、advantages、gaps、suggestions
```

## 后续接口，第一期不做

- `POST /api/resumes`：上传 PDF
- `POST /api/jobs`：保存 JD
- `POST /api/matches`：真实 AI 匹配
- `GET /api/reports`：历史报告
- `POST /api/auth/login`：登录
