# 轻量任务拆分

这份文档说明我们按什么顺序做。

原则：每一轮只做一个能看到效果的小闭环。

## 第 1 轮：前后端跑起来

目标：

```text
后端能启动，前端能启动，前端能确认后端在线
```

| 任务 | 负责人 | 完成标准 |
| --- | --- | --- |
| 初始化 Spring Boot 后端 | 余骏腾 | 后端本地能启动 |
| 写 `/api/health` | 余骏腾 | 浏览器或接口工具能访问 |
| 初始化前端项目 | 巫子萱 | 前端本地能打开 |
| 前端调用 `/api/health` | 巫子萱 | 页面能显示后端连接成功 |
| 本地跑通后端 | 古康杰 | 能启动后端并访问 health 接口 |

## 第 2 轮：Mock Demo

目标：

```text
输入简历内容和 JD，点击分析，页面展示一份固定报告
```

| 任务 | 负责人 | 完成标准 |
| --- | --- | --- |
| 写 `/api/matches/mock` | 余骏腾 | 返回固定匹配报告 |
| 做 Demo 页面 | 巫子萱 | 有简历输入框、JD 输入框、分析按钮 |
| 前端展示报告 | 巫子萱 | 展示分数、优势、短板、建议 |
| 熟悉后端结构 | 古康杰 | 能说清 controller/service 在哪里 |

## 第 3 轮：接真实能力

第一期完成后再做。

| 任务 | 负责人 | 完成标准 |
| --- | --- | --- |
| PDF 上传 | 古康杰 | 后端能接收 PDF 文件 |
| PDF 文本解析 | 古康杰 | 能提取 PDF 文本 |
| 接真实大模型 API | 余骏腾 | 能返回真实 AI 报告 |
| 前后端联调真实接口 | 全员 | 页面展示真实分析结果 |

## 第 4 轮：保存历史

真实 AI 跑通后再做。

| 任务 | 负责人 | 完成标准 |
| --- | --- | --- |
| 建 MySQL 表 | 余骏腾 | 能保存简历、JD、报告 |
| 保存历史报告 | 余骏腾 | 刷新后还能查到报告 |
| 历史记录页面 | 巫子萱 | 能查看之前的报告 |

## Todo 状态

飞书 Todo 只用 4 个状态：

- 待开始
- 进行中
- 卡住
- 已完成

优先级只用 2 个：

- P0：当前阶段必须做
- P1：有时间再做
