# 轻量项目启动说明

这份文档说明我们这个 3 人小团队怎么协作。

我们不搞复杂流程。当前目标是先做出一个能演示的 Demo，再逐步补真实能力。

## 团队分工

| 人员 | 角色 | 做什么 |
| --- | --- | --- |
| 余骏腾 | 负责人 / 后端 | 定第一期范围、拆任务、写后端接口、推进联调 |
| 古康杰 | 后端 | 跑通后端项目、熟悉结构、后续负责 PDF 或数据库模块 |
| 巫子萱 | 前端 | 做页面、调用接口、展示结果 |
| 刘科宇 | 产品顾问 | 看流程是否合理，帮忙砍掉不必要功能 |

## 协作方式

只用 3 个东西：

1. 飞书群：日常沟通、同步问题、确认决策。
2. 飞书 Todo：只放当前阶段任务。
3. Git 仓库：放代码和完整文档。

群里不刷长文档，只发摘要和关键决策。

## 当前阶段

当前阶段是第一期 Demo 骨架。

目标：

```text
前端页面 -> 调后端接口 -> 后端返回 mock 匹配报告 -> 前端展示结果
```

第一期先不做登录、数据库、PDF 解析、真实 AI、部署。

## 沟通规则

- 卡住超过 30 分钟就发群里。
- 接口变化必须同步到群里和接口文档。
- 每次只做一个小功能，做完先跑通。
- 不懂的代码先让 AI 解释，再合并。
